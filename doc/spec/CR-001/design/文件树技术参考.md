# 真正的“文件树拖拽排序”怎么做

如果你要的是像 VSCode / Finder 那样的效果，需要这几个步骤。

---

## 第一步：拍平树

```jsx
function flattenTree(nodes, parentId = null, depth = 0) {
  let result = [];

  for (const node of nodes) {
    result.push({
      id: node.id,
      name: node.name,
      type: node.type,
      parentId,
      depth,
      children: node.children || [],
    });

    if (node.children?.length) {
      result = result.concat(flattenTree(node.children, node.id, depth + 1));
    }
  }

  return result;
}
```

---

## 第二步：根据拍平结果重建树

```jsx
function buildTree(flattened) {
  const map = new Map();
  const root = [];

  flattened.forEach((item) => {
    map.set(item.id, { ...item, children: [] });
  });

  flattened.forEach((item) => {
    const node = map.get(item.id);

    if (item.parentId === null) {
      root.push(node);
    } else {
      const parent = map.get(item.parentId);
      if (parent) {
        parent.children.push(node);
      }
    }
  });

  return root;
}
```

---

## 第三步：移动节点时更新 parentId 和 depth

拖动时你通常会得到：

- `active.id`：被拖的节点
- `over.id`：目标节点

然后你需要决定：

- 它是排到目标节点前面？
- 成为目标节点的子节点？
- 还是与目标节点同级？

一般可以结合：

- 鼠标横向偏移量 => 决定层级深度
- 鼠标纵向位置 => 决定上下顺序

---

## 一个更接近真实树拖拽的数据处理思路

伪代码如下：

```js
function moveNode(flattened, activeId, overId, projectedDepth) {
  const oldIndex = flattened.findIndex((i) => i.id === activeId);
  const newIndex = flattened.findIndex((i) => i.id === overId);

  const newItems = arrayMove(flattened, oldIndex, newIndex);

  const item = newItems[newIndex];
  item.depth = projectedDepth;

  // 根据前一个节点推导 parentId
  if (projectedDepth === 0) {
    item.parentId = null;
  } else {
    for (let i = newIndex - 1; i >= 0; i--) {
      if (newItems[i].depth === projectedDepth - 1) {
        item.parentId = newItems[i].id;
        break;
      }
    }
  }

  return newItems;
}
```

# 实现时要注意的坑

## 1. 不能拖到自己的后代里

比如把 `src` 拖到 `src/components` 下面，这是非法的。

你需要在移动前判断：

```js
function isDescendant(items, parentId, targetId) {
  let current = items.find((i) => i.id === targetId);

  while (current) {
    if (current.parentId === parentId) return true;
    current = items.find((i) => i.id === current.parentId);
  }

  return false;
}
```

---

## 2. 文件和文件夹的规则不同

你可能希望：

- **文件夹**可以有子节点
- **文件**不能接收子节点

那就要在放置时限制目标类型。

---

## 3. 展开/折叠状态要单独存

不要混在树结构逻辑里，建议：

```js
const [expandedIds, setExpandedIds] = useState(new Set(['1']));
```

---

## 4. 大数据量时要考虑性能

节点很多时建议：

- `React.memo`
- 拍平结果 `useMemo`
- 避免每次拖动都深拷贝整棵树
- 必要时做虚拟列表

不过“树 + 拖拽 + 虚拟滚动”会复杂很多。
